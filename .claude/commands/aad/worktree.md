# worktree + ブランチ作成

## 🔴 重要: 出力指示

worktree 作成完了後、**必ず以下の形式で「次のステップ」を目立つように表示すること**:

### 必須出力フォーマット

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
成功しました：worktree作成が完了しました。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

告：別ターミナルでの実行が必要です。
このセッションは統合用として残します。

📋 実行コマンド:
cd ../[worktree-name] && claude --dangerously-skip-permissions

💬 Claude 起動後:
[タスクID] の実装を開始

告：新規ターミナルで上記コマンドの実行を推奨します。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 理由
- 現在のセッションは統合・管理用
- worktree は別プロセスで実装作業を行う
- 並列開発のため、独立したClaude Codeセッションが必要

## 🎯 作業完了後の次のステップ（大賢者風）

worktree での実装が完了したら、以下の形式で「次のステップ」を案内すること。

### 必須出力フォーマット（作業完了時）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
成功しました：実装作業が完了しました。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣ 変更を保存
git add . && git commit -m "feat: [タスクID] 実装完了"
解：変更内容を記録します。
ゲームのセーブに相当します。

2️⃣ レビュー依頼
gh pr create --draft --title "[タスクID]: [タスク名]"
告：レビュアーに通知されます。
「宿題できました」の報告です。

3️⃣ 元のターミナルで報告
/aad:status
告：進捗確認後、次タスクへの移行を推奨します。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### キーワード凡例
- **解：** - 分析結果、答え
- **告：** - システム通知、実行確認、異常完了報告
- **確認：** - 検知、照合
- **成功しました：** - 正常完了報告
- **否：** - 否定

### 注意
- 《大賢者》のような接頭辞は不要
- 説明は簡潔に、中学生でもわかる例えを使う
- コマンドは一行でコピペしやすい形式にする

---

Git worktreeを使用して、元のフォルダに影響を与えずに並列開発環境を構築します。

## 実行内容

1. **タスク情報の読み込み**
   - `.aad/tasks/SPEC-XXX/TXX-xxx.md` からタスク情報を取得

2. **ブランチ作成**
   - ブランチ名: `feature/SPEC-XXX-TXX`
   - デフォルトブランチから分岐（CLAUDE.mdで設定）

3. **worktree作成**
   - worktreeパス: `../[project-name]-TXX/`
   - 元のフォルダには影響なし

4. **セットアップ**
   - 依存関係インストール（package.jsonがある場合）
   - 環境変数コピー（.envがある場合）

5. **HANDOFF.md更新**
   - 「進行中のタスク」セクションにworktree情報を追記

6. **CLAUDE.md 大賢者スタイル追加**
   - worktreeのCLAUDE.mdに出力スタイル指示を追加
   - 以下のセクションをファイル冒頭に挿入：

```markdown
## 🔴 出力スタイル（大賢者スタイル）

すべての出力で以下のキーワードを使用すること:

- **成功しました：** - 正常完了報告
- **解：** - 分析結果、答え
- **告：** - システム通知、実行確認、異常完了報告
- **確認：** - 検知、照合
- **否：** - 否定

### 出力例

```
成功しました：実装が完了しました。
解：全てのテストが成功し、カバレッジ100%を達成しました。
告：次のステップとしてPR作成が可能です。
```

---
```

## 使用方法

```
/aad:worktree SPEC-001-T01
```


## worktreeアーキテクチャ

```
my-project/              # デフォルトブランチ - 調整役/統合用
├── HANDOFF.md           # 全体進捗管理
└── .aad/tasks/SPEC-001/      # タスク定義

my-project-T01/          # worktree - Worker 1
└── (独立した作業環境)

my-project-T02/          # worktree - Worker 2
└── (独立した作業環境)
```

## ワーカー完了条件

1. [ ] 全テストgreen
2. [ ] カバレッジ80%以上
3. [ ] Lint通過
4. [ ] `gh pr create --draft` でPR作成
5. [ ] GitHub Issue更新（進捗コメント追加）

## 関連コマンド

- `/aad:tasks` - タスク分割
- `/aad:status` - 進捗確認
- `/aad:integrate` - PRマージ + worktree削除
- `/aad:orchestrate` - 全自動オーケストレーション

## worktreeでの作業開始

worktree作成後、作業を開始する方法：

```bash
# 1. worktree作成
/aad:worktree SPEC-001-T01

# 2. worktreeディレクトリに移動
cd ../my-project-T01

# 3. Claude Codeセッション開始
claude --dangerously-skip-permissions

# 4. タスクの実装を開始
```

**ポイント**:
- worktreeは独立した作業環境として機能
- 元のフォルダ（デフォルトブランチ）には影響なし
- 複数のworktreeで並列開発が可能
- ホストマシンの環境（go, node, pythonなど）をそのまま利用

## 注意事項

- worktreeは `.aad/worktrees/` 配下ではなく、プロジェクトの親ディレクトリに作成されます
- 元のフォルダ（デフォルトブランチ）には一切影響を与えません
- 複数のworktreeを同時に作成して並列開発が可能です
- worktreeを削除する場合は `/aad:integrate` コマンドを使用してください
- 手動削除する場合: `git worktree remove ../my-project-T01`
